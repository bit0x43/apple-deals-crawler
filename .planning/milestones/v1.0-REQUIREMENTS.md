# Requirements: apple-deals-crawler — v1.0 Archive

## v1 Requirements

### Crawling

- [x] **CRAWL-01**: System crawls Apple Mac product prices from tiendasishop.com using Playwright
- [x] **CRAWL-02**: System crawls Apple Mac product prices from mac-center.com using Playwright
- [x] **CRAWL-03**: Each crawled record stores: reference, sku, memory, storage, color, price, url, source, crawled_at
- [x] **CRAWL-04**: System skips writing a record when the price is unchanged from the previous crawl (change-only storage)
- [x] **CRAWL-05**: GitHub Actions cron runs the crawler daily on a schedule

### Database

- [x] **DB-01**: System uses SQLite as the default backend (zero-config, works out of the box)
- [x] **DB-02**: System switches to PostgreSQL when DATABASE_URL env var is set (Neon serverless as primary free host)
- [x] **DB-03**: System auto-prunes records older than configurable retention period (default: 90 days)
- [x] **DB-04**: User can run `apple-deals db clean` to manually trigger pruning with optional --dry-run preview
- [x] **DB-05**: User can run `apple-deals db stats` to view row count, DB size, oldest/newest record, and estimated growth rate

### TUI

- [x] **TUI-01**: User can open an interactive TUI via `apple-deals tui`
- [x] **TUI-02**: TUI catalog view shows current prices filterable by model and store, sortable by price
- [x] **TUI-03**: TUI history view shows price trends per product over the rolling window
- [x] **TUI-04**: User can toggle between catalog and history views (tab or keyboard shortcut)

### Alerts

- [x] **ALRT-01**: System sends a Telegram message when a product's price drops below a configurable threshold (% or absolute amount)
- [x] **ALRT-02**: Alert threshold and Telegram credentials are configurable via environment variables or config file

### CLI

- [x] **CLI-01**: User can run `apple-deals crawl` to trigger a manual crawl
- [x] **CLI-02**: User can run `apple-deals tui` to open the interactive terminal UI
- [x] **CLI-03**: CLI provides --help for all commands with clear descriptions

### Documentation

- [x] **DOCS-01**: Project has MkDocs + Material docs site deployable to GitHub Pages
- [x] **DOCS-02**: Docs cover installation, CLI reference, configuration, and self-hosting guide
- [x] **DOCS-03**: GitHub Actions workflow deploys docs to GitHub Pages on push to main

### Dev Practices

- [x] **DEV-01**: Pre-commit hooks enforce secret scanning (detect-secrets)
- [x] **DEV-02**: Pre-commit hooks enforce ruff for linting and formatting
- [x] **DEV-03**: Pre-commit hooks enforce mypy for type checking
- [x] **DEV-04**: Pre-commit hooks enforce trailing whitespace and file hygiene checks
- [x] **DEV-05**: Docker + docker-compose setup for local dev and optional self-hosted deployment
- [x] **DEV-06**: npx autoskills installed after stack is finalized — **Deferred** (requires gsd-sdk global setup)

### Phase 8 Extras (Post-v1 Plan)

- [x] **STOCK-01**: Track product stock status (in_stock column, Shopify `variants[0].available`)
- [x] **STOCK-02**: Enrich RAM from product pages via Playwright browser context
- [x] **STOCK-03**: Filter out products with < 16GB RAM
- [x] **STOCK-04**: High-memory Telegram alert (>= 24GB, configurable via HIGH_MEMORY_THRESHOLD)
- [x] **STOCK-05**: TUI catalog shows stock status (In Stock / Out of Stock)
