## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**

- Change-only storage (dedup) — keeps DB size manageable; 20 products × 2 stores rarely change daily
- Neon as default Postgres host — serverless auto-suspend = $0 idle cost; daily cron cold starts work fine
- SQLite default, Postgres via env var — zero-config local dev; same models target both via SQLAlchemy
- 90-day rolling window — covers seasonal price patterns; keeps Neon free tier safe
- Threshold-based alerts (not any-drop) — minor daily fluctuations shouldn't page the user
- MkDocs Material for docs — Python-native, excellent for CLI tools, trivial GitHub Pages deploy
- Pre-commit hooks from day 1 — secret scanning + ruff + mypy enforced early
- RAM enrichment via Playwright browser context — `data-title` attrs, swatch labels, embedded JSON
- Stock tracked via `in_stock` column (mirrors Shopify's `variants[0].available`)
- Products with < 16GB RAM excluded from storage

**Issues Resolved:**
