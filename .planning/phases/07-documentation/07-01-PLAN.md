Step 1: Create .github/workflows/ directory and docs.yml per D-08.

```yaml
name: docs

on:
  push:
    branches:
      - main
    paths:
      - docs/**
      - mkdocs.yml
      - pyproject.toml
  workflow_dispatch:

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Configure Git Credentials
        run: |
          git config user.name github-actions[bot]
          git config user.email 41898282+github-actions[bot]@users.noreply.github.com

      - uses: actions/setup-python@v5
        with:
          python-version: "3.13"

      - name: Install uv
        run: pip install uv

      - name: Install dependencies
        run: uv sync --group dev

      - name: Build and deploy docs
        run: uv run mkdocs gh-deploy --force
```

Notes on decisions (per D-08 and RESEARCH.md):
- `paths` filter prevents unnecessary deploys when only source code changes (RESEARCH recommendation on Open Question 2)
- `workflow_dispatch` allows manual triggers
- `fetch-depth: 0` ensures correct git history for gh-deploy
- `uv sync --group dev` installs the full project so mkdocs-typer2 can import the Typer app at build time (Pitfall 6 mitigation)
- --force flag avoids push conflicts (Pitfall 5 mitigation)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions → gh-pages branch | Workflow with `contents: write` permission pushes build output to gh-pages. |
| Doc content → consumer | Static HTML served via GitHub Pages — no form input, dynamic rendering, or user-submitted content. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | .github/workflows/docs.yml | mitigate | Workflow only triggers on push to main with path filtering. `contents: write` is scoped to the docs deploy job. Branch protection on main prevents unauthorized pushes. |
| T-07-02 | Spoofing | GitHub Pages hosted site | accept | Static site has no authentication or user input. The public nature of GitHub Pages is intentional for an OSS project. No spoofing surface exists. |
| T-07-03 | Repudiation | mkdocs-typer2 module import | accept | The plugin imports the Typer app at build time. If the module is compromised, malicious code executes during build. Mitigation: uv lockfile pins dependency versions; CI runs from clean checkout. |
</threat_model>
