
**If the run fails**, diagnose and fix using these rules:

**detect-secrets fails ("Potential secrets baseline not found"):**
- Re-run: `uv run detect-secrets scan > .secrets.baseline`
- Then re-run pre-commit

**detect-secrets fails ("Secrets found"):**
- Inspect the flagged file: `cat .secrets.baseline` to see what was flagged
- If it is a false positive (e.g., a test database URL like `sqlite:///:memory:`), update the baseline:
  `uv run detect-secrets scan --update .secrets.baseline`
- Then re-run pre-commit

**ruff-check fails:**
- ruff auto-fixes with `--fix` flag already in config
- If still failing after auto-fix: `uv run ruff check src/ tests/ --fix` and inspect remaining errors
- Common cause: unused imports in `__init__.py` stubs — if an `__init__.py` is empty, ruff should not flag it; if it has imports, check they are actually used

**ruff-format fails:**
- ruff-format rewrites files in-place automatically; re-running should clear the failure

**mypy fails:**
- Must show zero errors. Common causes in this codebase:
  - "Cannot find implementation or library stub for module named 'sqlalchemy'" → verify `additional_dependencies` in `.pre-commit-config.yaml` has `sqlalchemy[mypy]>=2.0.49`
  - "error: Cannot determine type of..." on ORM columns → verify `plugins = ["sqlalchemy.ext.mypy.plugin"]` is in `[tool.mypy]` in pyproject.toml
  - Missing return type on a function → add `-> None:` or correct return type
- If mypy fails in pre-commit but passes locally (`uv run mypy src/`), the issue is missing `additional_dependencies` in the hook config

**trailing-whitespace / end-of-file-fixer fail:**
- These hooks auto-fix in-place; re-running should clear them

**check-added-large-files fails:**
- A file >500KB was staged. Identify it and add it to `.gitignore` if it should not be tracked (e.g., `apple_deals.db`)

After all fixes, run once more to confirm everything is green:
```bash
cd . && uv run pre-commit run --all-files
```

The final run must exit 0.
