```dockerfile
# Dockerfile
# Production image: python:3.13-slim + uv + Playwright Chromium
#
# Source patterns:
#   - docs.astral.sh/uv/guides/integration/docker/ (COPY --from, cached layers)
#   - playwright.dev/python/docs/docker (--with-deps, --ipc=host)
#
# Usage:
#   docker build -t apple-deals-crawler .
#   docker run --env-file .env apple-deals-crawler crawl

FROM python:3.13-slim

# Install uv binary from official distroless image (no curl/pip needed)
COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /bin/

WORKDIR /app

# Install system dependencies needed for Playwright's apt-get calls
# (playwright install --with-deps handles the actual browser deps)
RUN apt-get update && apt-get install -y --no-install-recommends \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*

# Install project dependencies (cached layer — invalidated only when lockfile/toml change)
RUN --mount=type=cache,target=/root/.cache/uv \
    --mount=type=bind,source=uv.lock,target=uv.lock \
    --mount=type=bind,source=pyproject.toml,target=pyproject.toml \
    uv sync --locked --no-dev --no-install-project

# Copy project source
COPY . /app

# Install the project package and remaining dependencies
RUN --mount=type=cache,target=/root/.cache/uv \
    uv sync --locked --no-dev

# Install Playwright Chromium browser with system dependencies
RUN python -m playwright install --with-deps chromium

ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1

ENTRYPOINT ["uv", "run", "apple-deals"]
CMD ["crawl"]
```

Create `docker-compose.yml` at project root:

```yaml
# docker-compose.yml
# Usage:
#   docker-compose up                   # Start crawler (will exit after crawl)
#   docker-compose --profile with-db up -d postgres  # Start local PG for development
#
# Env vars are loaded from .env file (not committed to git).
# Copy .env.example to .env and fill in values.

services:
  crawler:
    build: .
    env_file:
      - .env
    # Required for Chromium shared memory in Docker
    ipc: host
    init: true
    depends_on:
      postgres:
        condition: service_healthy
        required: false

  postgres:
    image: postgres:16-alpine
    profiles:
      - with-db
    env_file:
      - .env
    environment:
      POSTGRES_USER: ${POSTGRES_USER:-crawler}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-crawler}
      POSTGRES_DB: ${POSTGRES_DB:-apple_deals}
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "127.0.0.1:5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-crawler} -d ${POSTGRES_DB:-apple_deals}"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  pgdata:
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker container → Host filesystem | .env file mounted at runtime via env_file in docker-compose |
| Docker container → External services | DATABASE_URL connects to Neon, TELEGRAM_BOT_TOKEN connects to Telegram API |
| Docker build → pip/PyPI | uv sync fetches packages from PyPI during build; pinned versions via uv.lock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Information Disclosure | Secrets in Docker image | mitigate | Secrets are never baked into the image. Dockerfile reads env vars only at runtime via `--env-file .env`. The .dockerignore excludes .env from the build context. |
| T-06-06 | Tampering | Base image integrity | mitigate | `python:3.13-slim` is an official Docker image signed by Docker. `ghcr.io/astral-sh/uv` images have SLSA provenance attestations. For production hardening, pin by digest instead of tag. |
| T-06-07 | Tampering | Dependency integrity | mitigate | `uv sync --locked` enforces exact transitive dependency hashes from uv.lock. Any mismatch between lockfile and registry fails the build. |
| T-06-08 | Denial of Service | Chromium OOM in Docker | mitigate | Dockerfile + docker-compose configure `--ipc=host` and `init: true` to prevent Chromium shared memory crashes. Timeout set at workflow level in GitHub Actions. |
| T-06-09 | Elevation of Privilege | Docker socket mount | accept | docker-compose does NOT mount `/var/run/docker.sock`. Container runs with default Docker capabilities (no --privileged). No container escape vector introduced. |
</threat_model>
