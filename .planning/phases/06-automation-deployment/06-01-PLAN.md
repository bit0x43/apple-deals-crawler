```yaml
# .github/workflows/crawl.yml
# Daily cron job: crawls Apple Mac prices from Colombian retailers.
# Runs every day at 8:00 AM UTC (3:00 AM Bogota standard time).
#
# Secrets required (set in GitHub repository settings):
#   DATABASE_URL        — PostgreSQL (Neon) or SQLite default
#   TELEGRAM_BOT_TOKEN  — Telegram bot token for price alerts
#   TELEGRAM_CHAT_ID    — Telegram chat ID for alert recipient
#
# References:
#   - docs.astral.sh/uv/guides/integration/github/ (setup-uv)
#   - playwright.dev/python/docs/ci (playwright install --with-deps)

name: Daily Crawl

on:
  schedule:
    - cron: '0 8 * * *'
  workflow_dispatch:

# Minimal permissions — only read contents, no write access
permissions:
  contents: read

jobs:
  crawl:
    runs-on: ubuntu-latest
    timeout-minutes: 30

    steps:
      - uses: actions/checkout@v6

      - name: Install uv
        uses: astral-sh/setup-uv@v3
        with:
          enable-cache: true
          cache-dependency-glob: "uv.lock"

      - name: Set up Python 3.13
        run: uv python install 3.13

      - name: Install project dependencies
        run: uv sync --locked --no-dev

      - name: Install Playwright Chromium
        run: python -m playwright install --with-deps chromium

      - name: Run crawl
        run: uv run apple-deals crawl
        env:
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
          TELEGRAM_BOT_TOKEN: ${{ secrets.TELEGRAM_BOT_TOKEN }}
          TELEGRAM_CHAT_ID: ${{ secrets.TELEGRAM_CHAT_ID }}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Runner → GitHub Secrets | Secrets decrypted at runtime by GitHub Actions, mapped to env vars via `${{ secrets.* }}` |
| Runner → External Services | DATABASE_URL connects to Neon (Postgres), TELEGRAM_BOT_TOKEN connects to Telegram API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Secrets in CI logs | mitigate | Secrets referenced via `${{ secrets.* }}` in `env:` blocks only — never in `run:` command strings. GitHub Actions auto-masks secret values in logs. No echo or debug logging of env vars. |
| T-06-02 | Tampering | Third-party GitHub Actions | mitigate | `actions/checkout@v6` and `astral-sh/setup-uv@v3` pinned by major version tag. For production hardening, switch to commit SHA pinning (e.g., `@e6d535b`). Current pinning by major version allows patch updates. |
| T-06-03 | Elevation of Privilege | Workflow permissions | mitigate | Explicit `permissions: contents: read` restricts token to read-only for the repo. No write access to secrets, code, or deployments. |
| T-06-04 | Tampering | Supply chain (uv.lock) | mitigate | `uv sync --locked` enforces exact dependency versions from lockfile. Any mismatch between pyproject.toml and uv.lock causes CI failure (fail-fast). |
</threat_model>
