- `.planning/phases/10-tui-filters-ordering/10-RESEARCH.md` — Pattern 1: DataTable Sort with Custom Key, Pattern 3: Sort State Management
</read_first>

<action>
Replace the existing sort handling in `CatalogScreen`:

1. Add class attributes: `_sort_column: str | None = None`, `_sort_reverse: bool = False`
2. Replace `on_data_table_header_selected()` to handle memory and storage columns:
   - If clicked column is `"price"`, `"memory"`, or `"storage"`:
     - If same column as `_sort_column`, toggle `_sort_reverse`
     - Else set `_sort_column` to clicked column, reset `_sort_reverse` to `False`
     - Call `self._apply_sort_to_table()`
3. Add `_apply_sort_to_table()` method:
   - If `_sort_column == "memory"`: `table.sort("memory", key=_memory_gb, reverse=self._sort_reverse)`
   - If `_sort_column == "storage"`: `table.sort("storage", key=_storage_bytes, reverse=self._sort_reverse)`
   - If `_sort_column == "price"`: `table.sort("price", key=lambda p: int(p.replace(",", "")), reverse=self._sort_reverse)`
   - If `_sort_column` is None: do nothing (default sort from `_populate_table` applies)
4. Remove the old price-only `_sort_reverse` field declaration (move to the new class-level attribute)
</action>

- Clicking Price column header sorts by price numerically (not lexicographically)
- Second click reverses sort direction per column
- Switching active column resets direction to ascending
